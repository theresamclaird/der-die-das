# der · die · das — A1 article trainer (Phase 0)

A local-first PWA for drilling German noun genders with **implicit grading**:
one committed tap is your answer *and* your grade. Wrong → `Again`; correct →
`Hard/Good/Easy` inferred from normalized response latency. Scheduling is real
[FSRS](https://github.com/open-spaced-repetition/ts-fsrs); progress persists in
IndexedDB. This implements Phase 0 of `IMPLEMENTATION_PLAN.md`.

## Run

```bash
npm install
npm run dev        # http://localhost:5173
npm test           # unit tests for the inference layer
npm run build      # production build (PWA)
npm run preview    # serve the build; service worker active here
```

Node 18+ recommended.

## How it maps to the design docs

| File | Role | Design ref |
|---|---|---|
| `src/lib/scheduler.js` | ts-fsrs wrapper (emptyCard / applyRating / isDue / preview) | DESIGN §4.2 |
| `src/lib/inference.js` | implicit grading: latency → Again/Hard/Good/Easy | DESIGN §4.6 |
| `src/lib/db.js` | IndexedDB: card state, append-only review log, meta | DESIGN §4.5, §6 |
| `src/lib/session.js` | due selection + new-card budget + in-session re-show | DESIGN §7 |
| `src/App.jsx` | the review loop wiring it together | IMPL Phase 0 |
| `src/data/nouns.json` | content (output of the `pipeline/`) | DESIGN §3 |

The scheduler is deliberately the only file that imports ts-fsrs — swapping
schedulers later touches just that module. The inference layer is framework-free
and unit-tested.

## Calibration

The latency thresholds in `inference.js` (`TUNING`) are starting guesses, not
truths. Every answer logs raw `latency_ms` / `recall_ms` to IndexedDB, so after
a couple of weeks of real use you can pull the log and tune `EASY_RATIO`,
`HARD_RATIO`, `READ_BASE`, and `READ_PER_CHAR` against your own data instead of
trusting the defaults. The review log is the safety net: no information is lost
by getting the thresholds slightly wrong now.

## What's stubbed in Phase 0

- **Icons.** `public/manifest.webmanifest` references `/icons/icon-192.png`,
  `icon-512.png`, and `icon-512-maskable.png`. Add real PNGs at `public/icons/`
  before installing to a phone home screen.
- **Service worker** is a minimal runtime cache (offline after first load). For
  production-grade precaching/versioning, switch to `vite-plugin-pwa`.
- **No auth / no cloud sync.** That's Phase 1 (Cognito + DynamoDB). Progress is
  per-device until then. `db.js` already writes a review log shaped for later
  sync and FSRS re-optimization.

## Content

`src/data/nouns.json` is generated by the separate `pipeline/` (Goethe-leveled
lemmas joined against `german-nouns` for gender + plural). To refresh or extend
to A2/B1, regenerate there and copy the output here.

## Notes on the interaction

- The question never shows a live timer — that would change how fast you answer.
  Timing is revealed only after you commit.
- A correct-but-distracted answer (tab hidden, or > 12 s) has its timing
  discarded rather than scored as a lapse.
- The override row exists for the rare case the inference misreads you; overrides
  are flagged in the log so they can inform later threshold tuning.
